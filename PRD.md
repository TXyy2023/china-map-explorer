# AI 传统文化交互地图 PRD

## 1. 产品定位

本项目是一个面向比赛展示的 AI 传统文化交互地图 Demo。产品通过可视化地图、AI 图片/视频素材和可配置交互，让学生和评委以新奇、可浏览、可探索的方式了解中国传统文化。

第一版重点不是固定某一个文化主题，而是搭建一套可配置的文化地图框架。具体展示什么文化、使用什么图片风格、采用什么内容文案，都应尽量通过配置或开发模式添加，避免写死在代码中。

## 2. 目标用户

- 学生：通过地图和互动内容快速理解中国传统文化的地域分布与差异。
- 比赛评委：在短时间内看到产品的视觉表现、交互创新、AI 内容生产能力和可扩展性。

## 3. 核心体验

用户进入页面后，先选择一个文化主题，再进入对应的中国地图视图。地图不只是行政区划展示，而是根据当前文化主题，把多个省份归为不同的文化区。

用户可以像进入省份一样进入某个文化区，查看该文化区的 AI 图片、文字说明、视频内容和可交互内容。文化区内部的展示形式、素材、动效和小游戏入口都应支持配置。

## 4. MVP 范围

第一版需要完成以下能力：

- 支持文化主题配置，而不是把具体文化内容写死。
- 支持从文化主题进入中国地图。
- 地图层级升级为：全国大盘 -> 美食大区 -> 省份下钻多级交互。
- 支持把多个省份配置为同一个文化区，并在进入大区后支持区内各省独立下钻交互与特定答题。
- 支持文化区点击/进入后的详情展示。
- 支持 AI 图片、文字、视频等素材类型。
- 支持预留小游戏或互动模块入口。
- 支持开发模式和运行模式。
- 支持通过配置添加交互函数和入口函数。

第一版暂不强制完成：

- 市/区县级下钻。
- 手绘任意不规则文化边界。
- 完整小游戏系统。
- 固定文化主题库。
- 固定 AI 图片风格。
- 完整内容管理后台。

## 5. 页面与模式

### 5.1 主题选择页

主题选择页用于展示当前可用的文化主题。每个主题由配置生成，至少包含：

- 主题名称
- 主题简介
- 封面素材
- 进入地图的入口函数

点击主题后，执行该主题配置的入口函数，进入对应文化地图。

### 5.2 运行模式

运行模式是普通用户和评委看到的浏览模式。

运行模式能力：

- 查看文化主题。
- 进入文化地图。
- 点击或进入文化区。
- 浏览 AI 图片、文字和视频内容。
- 触发配置好的 hover、click、enter 等交互函数。
- 进入预留的互动小游戏模块。

运行模式不允许直接编辑配置或素材。

### 5.3 开发模式

开发模式用于快速搭建和调整展示内容。

开发模式能力：

- 新建、编辑、删除文化主题。
- 配置文化区与省份的归属关系。
- 添加 AI 图片、视频、文字等素材。
- 调整素材在地图或详情页中的位置、大小、层级和显示方式。
- 为文化区或素材绑定交互函数。
- 配置主题入口函数。
- 预览运行模式效果。
- 保存或导出文化主题配置。

## 6. 地图与文化区规则

地图底层使用中国省级行政区划。文化展示层在行政区划之上运行，通过配置把一个或多个省份归入同一个文化区。

示例规则：

```json
{
  "id": "culture-area-001",
  "name": "示例文化区",
  "provinceAdcodes": ["110000", "120000", "130000"],
  "style": {
    "fill": "#d49235",
    "stroke": "#ffffff"
  }
}
```

同一个文化区内的省份可以使用统一颜色、统一 hover 效果和统一点击逻辑。点击文化区时，交互体验应类似当前地图进入省份的体验，但进入的是文化区详情，而不是行政下一级地图。

## 7. 配置模型

### 7.1 文化主题配置

```json
{
  "id": "theme-id",
  "name": "文化主题名称",
  "description": "主题说明",
  "coverAssetId": "asset-cover",
  "entryAction": "enterThemeMap",
  "areas": [],
  "assets": [],
  "interactions": []
}
```

### 7.2 文化区配置

```json
{
  "id": "area-id",
  "name": "文化区名称",
  "description": "文化区说明",
  "provinceAdcodes": [],
  "assets": [],
  "detailLayout": "default",
  "interactions": {
    "hover": "showAreaPreview",
    "click": "enterAreaDetail",
    "enter": "playEnterAnimation"
  }
}
```

### 7.3 素材配置

```json
{
  "id": "asset-id",
  "type": "image",
  "title": "素材标题",
  "description": "素材说明",
  "src": "素材地址",
  "position": {
    "x": 0.5,
    "y": 0.5
  },
  "size": {
    "width": 320,
    "height": 220
  },
  "interactions": {
    "hover": "liftAsset",
    "click": "openAssetDetail"
  }
}
```

素材类型第一版至少支持：

- image
- video
- text
- game-entry

## 8. 交互函数设计

系统需要预留一套常用交互函数机制。文化主题、文化区和素材都可以绑定函数。

基础函数类型：

- hover：鼠标进入或聚焦时触发。
- leave：鼠标离开时触发。
- click：单击时触发。
- doubleClick：双击时触发。
- enter：进入文化区或详情时触发。
- exit：退出文化区或详情时触发。

第一版内置函数建议：

- enterThemeMap：进入指定文化主题地图。
- enterAreaDetail：进入文化区详情。
- showAreaPreview：显示文化区浮层预览。
- playEnterAnimation：播放进入动效。
- openAssetDetail：打开素材详情。
- playVideo：播放视频。
- openGameEntry：打开小游戏入口。

函数系统需要允许后续继续扩展，不要求第一版实现复杂脚本编辑器。

## 9. AI 与 MCP 接入

AI/MCP 主要服务于开发模式，用于提高内容生产效率。

第一版可规划以下能力：

- 根据文化区名称和描述生成 AI 图片提示词。
- 根据提示词生成或替换 AI 图片。
- 根据素材描述生成说明文字。
- 根据文化主题生成一组素材草案。
- 将生成结果写入当前文化主题配置。

AI 生成内容不应强依赖固定风格。图片风格、文案风格和视频风格应作为配置项或生成参数存在。

## 10. 详情页与互动模块

点击或进入文化区后，展示文化区详情。

详情页基础内容：

- 文化区名称
- 文化区说明
- 关联省份
- AI 图片列表
- 视频内容
- 文字内容
- 互动模块入口

互动模块第一版只需要预留入口和配置结构，具体游戏可以后续按需求补充。

可支持的互动形式包括：

- 地区与文化内容匹配
- 猜文化内容所属地区
- 拖拽素材完成任务
- 点击地图收集图鉴
- 简单问答闯关

## 11. 非目标范围

以下内容不作为第一版必须完成目标：

- 完整商业化 CMS。
- 大规模文化内容数据库。
- 用户登录和权限系统。
- 多人协作编辑。
- 精确到城市/区县的文化区编辑。
- 任意形状地图绘制器。
- 完整小游戏编辑器。

## 12. 验收标准

第一版完成后，应满足以下标准：

- 页面可以展示至少一个由配置生成的文化主题。
- 点击文化主题后，可以进入对应的中国地图。
- 地图可以展示省级行政区划。
- 可以通过配置把多个省份归为一个文化区。
- 同一文化区在地图上有统一的视觉反馈。
- 点击文化区后，可以进入文化区详情。
- 详情页可以展示 AI 图片、文字和视频内容。
- 详情页可以展示小游戏或互动模块入口。
- 开发模式可以新增或修改主题、文化区和素材配置。
- 运行模式不能直接编辑内容。
- 交互函数可以通过配置绑定到主题、文化区或素材。
- 主题、区域、素材和交互逻辑不依赖硬编码文化内容。

## 13. 后续版本规划

后续可以继续扩展：

- 市级、区县级地图下钻。
- 自定义文化区边界绘制。
- 更完整的开发模式编辑器。
- AI 视频生成和批量素材生成。
- 可视化函数编排。
- 完整小游戏模块。
- 多文化主题模板库。
- 本地配置导入导出。
- 在线发布与分享。
